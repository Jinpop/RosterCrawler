# RosterCrawler — KAU 종합정보시스템 수강생 명단 자동 수집기

한국항공대학교 종합정보시스템(nportal)의 **안전교육원 수강생 명단**을 자동으로 크롤링해
하나의 엑셀 파일(`KAU_MASTER_DATA.xlsx`)로 누적·정리하는 Python 자동화 도구입니다.

월별로 수십 개 과정의 명단을 일일이 조회·인쇄·복사하던 반복 업무를,
**로그인부터 엑셀 저장까지 전 과정 무인 자동화**로 바꿨습니다.

---

## 📌 문제 정의 (Motivation)

안전교육원은 매월 다수의 교육 과정을 운영하며, 각 과정마다 수강생 명단을
종합정보시스템에서 **과정 선택 → 인쇄(리포트) → 화면의 표를 눈으로 옮겨 적기** 방식으로
관리해야 했습니다.

- 과정 1건당 수 분, 한 달이면 수십 건 → **반복적이고 시간 소모가 큰 수작업**
- 리포트 뷰어(Crownix)는 표가 아닌 **좌표 기반으로 흩뿌려진 텍스트**라 단순 복사가 어려움
- 누락·오타 발생 가능성

이 프로젝트는 위 과정을 코드로 대체합니다.

---

## ✨ 주요 기능

- 🔐 **자동 로그인** — `.env`의 계정 정보로 로그인 폼을 자동 탐지·입력 (필드 ID를 몰라도 동작)
- 🧭 **메뉴 자동 이동** — `일반행정 → 부서업무 → 안전교육원 → 수강생관리`를 글자 기반으로 자동 클릭 (프레임 자동 탐색)
- 📅 **기간 일괄 수집** — 지정한 시작~종료 연·월의 모든 과정을 순회 수집
- 📐 **좌표 기반 표 복원** — Crownix 리포트의 흩어진 텍스트를 `top`/`left` 좌표로 행·열을 재구성
- 📄 **다중 페이지 처리** — 과정별 여러 페이지를 자동 페이지네이션
- 💾 **실시간 누적 백업** — 페이지 단위로 엑셀 저장, 중단되어도 진행분 보존 + 재실행 시 이어쓰기
- ⚙️ **환경변수 설정** — 헤드리스 모드 / 수집 기간을 `.env`로 제어

## 🚀 설계에서 짚은 문제와 해결

처음 동작하는 버전을 만들고 실제로 수백 명 규모를 돌려보면서, 무인 자동화를 막거나
속도를 떨어뜨리는 지점들을 직접 찾아 다음과 같이 해결했습니다.

| 짚은 문제 | 원인 | 해결 |
|------|--------|-------|
| 사람이 붙어 있어야 함 | 로그인·화면 이동이 수동 | `.env` 계정 자동 로그인 + 메뉴 글자 기반 자동 클릭 |
| 수집할수록 느려짐 | **수강생 1명마다** 누적 전체를 엑셀로 재작성 → 디스크 쓰기 O(n²) | **페이지 단위** 저장으로 변경 |
| 불필요하게 오래 대기 | 고정 `time.sleep` 남발 (4·5·2·1.5초…) | 조건 충족 시 즉시 진행하는 `WebDriverWait` |
| 서버에서 못 돌림 | GUI 창 고정 | **헤드리스 토글** 지원 |
| 기간 바꿀 때마다 코드 수정 | 수집 기간 하드코딩 | `.env`로 외부화 |

> 가장 큰 병목은 *수강생 한 명을 수집할 때마다 누적된 전체 데이터로 엑셀 파일을
> 통째로 다시 쓰던 로직*이었습니다. "느린 게 네트워크 탓"이라고 넘기지 않고
> 어디서 시간이 쌓이는지 직접 추적해, 저장 단위를 페이지로 바꿔 디스크 I/O를 수백 배 줄였습니다.

## 🛠 기술 스택

- **Python 3**
- **Selenium** — 브라우저 자동화 (로그인·메뉴·페이지네이션)
- **pandas / openpyxl** — 엑셀 입출력 및 누적 병합
- **webdriver-manager** — ChromeDriver 자동 설치/버전 관리
- **python-dotenv** — 계정·설정 환경변수 관리

## ⚙️ 설치 & 실행

### 1. 의존성 설치
```bash
pip install -r requirements.txt
```

### 2. 환경변수 설정
`.env.example`을 복사해 `.env`를 만들고 본인 정보로 채웁니다.
```bash
cp .env.example .env
```
```dotenv
id=학번/교직원번호
pass=비밀번호

HEADLESS=false      # true → 브라우저 창 없이 실행

START_YEAR=2024     # 수집 시작 연월
START_MONTH=11
END_YEAR=2025       # 수집 종료 연월 (포함)
END_MONTH=5
```

### 3. 실행
```bash
python main.py
```
완료되면 바탕화면에 `KAU_MASTER_DATA.xlsx`가 생성되고 폴더가 자동으로 열립니다.

> 💡 **처음엔 `HEADLESS=false`로 실행**해 메뉴 자동 클릭(`✓ '일반행정' 클릭` …)이
> 정상 동작하는지 확인한 뒤 헤드리스로 전환하는 것을 권장합니다.

## 🧩 동작 방식

```
[로그인] .env 계정 → 비밀번호 칸 자동 탐지 → 입력·제출
   ↓
[메뉴 이동] 일반행정 → 부서업무 → 안전교육원 → 수강생관리 (글자 기반 자동 클릭)
   ↓
[기간 순회] START~END 의 (연,월) 마다
   ├─ 검색 조건 설정 후 조회
   └─ 과정 목록 순회
        ├─ 과정 클릭 → 인쇄(리포트 새 창)
        ├─ Crownix 리포트의 텍스트 div 들을 top/left 좌표로 수집
        ├─ top 으로 행 묶기 → left 로 열 배치 → 한 행 복원
        ├─ 다음 페이지 버튼으로 페이지네이션
        └─ 페이지마다 엑셀에 누적 저장
   ↓
[완료] KAU_MASTER_DATA.xlsx
```

## 🩹 문제 해결 (Troubleshooting)

- **메뉴를 못 찾아요 (`⚠️ … 못 찾았습니다`)** — 실제 메뉴 글자가 코드와 다를 수 있습니다(띄어쓰기 등). `main.py`의 `MENU_PATH`를 화면 표기와 똑같이 맞추세요.
- **로그인 폼을 못 찾아요** — `main.py`의 `LOGIN_ID_SEL` / `LOGIN_PW_SEL`에 입력칸 CSS 선택자를 직접 지정하면 됩니다.
- **헤드리스에서 열이 어긋나요** — 좌표 기반 파싱이 렌더링 차이에 영향받을 수 있습니다. `HEADLESS=false`로 사용하세요.
- **ChromeDriver 오류** — `webdriver-manager`가 자동 관리하지만, Chrome 브라우저 자체는 최신으로 유지해야 합니다.
